# 새 Skill 생성 상세 가이드

## Step 1: 정보 수집

```
🎯 어떤 Skill을 만들까요?

1. **Skill 이름**: 수행하는 작업 (예: "PR 생성", "버그 수정")
2. **해결 문제**: 이 Skill 없으면 어떤 반복 작업 발생?
3. **기대 결과물**: 실행하면 무엇이 생성?
4. **참고 Skill**: 비슷한 기존 Skill? (선택)
```

---

## Step 2: 모호성 해결 (핵심)

> 참고: `clarifying-requirements` Skill의 패턴 활용

### Skill 생성 모호성 카테고리

- **트리거**: 어떤 말을 했을 때 이 Skill이 실행되어야 하나요? (한국어/영어)
- **범위**: 무엇이 포함? 명시적으로 제외되는 것은?
- **출력물**: 실행 결과 무엇이 생성? (파일, 문서, 코드?)
- **다른 Skill 구분**: 비슷한 기존 Skill과 어떻게 다른가요?
- **성공 기준**: 언제 "완료"라고 판단하나요?

### 질문 설계 원칙

- **구체적으로**: 추상적 질문 피하기 (❌ "어떤 기능?" ✅ "트리거 키워드는?")
- **선택지 제공**: 2-4개 옵션 제시 (❌ "어떻게?" ✅ "A, B 중 어떤 게?")
- **한 번에 하나**: 여러 질문 묶지 않기 (❌ "트리거와 출력은?" ✅ "트리거는?")

---

## Step 3: 이해 확인 (필수)

**사용자 승인 전까지 진행 금지:**

```
✅ 이렇게 이해했습니다:

**핵심 기능:**
- [기능 A]: [설명]

**제외:**
- [불필요한 것]

**트리거 조건:**
- "[한국어]", "[영어]"

**다른 Skill과 구분:**
- [구분 기준]: For X, use Y instead

**성공 기준:**
- [완료 조건]

맞나요?
```

### AI 체크리스트

- [ ] 모호성 카테고리 모두 확인했는가?
- [ ] 핵심 기능만 포함했는가? (오버스펙 금지)
- [ ] 다른 Skill과 명확히 구분했는가?

## Step 4: SKILL.md 작성

### 파일 구조

```
skills/{skill-name}/
├── SKILL.md              # 필수
└── references/           # 선택 (순차적 단계 있을 때)
```

### 템플릿

```markdown
---
name: {skill-name}
description: {무엇}. Use when user says "{한국어}", "{영어}". For X, use Y instead.
---

# {제목}

## 목표

{해결하는 문제}

## 언제 사용하는가?

### ✅ 사용 시점

- {조건 1}

### ❌ 다른 Skill 사용

- {조건} → {다른 Skill}

## AI 수행 절차

### Step 0: To-Do 생성

스킬 시작 시 아래 To-Do를 생성하여 진행 상황을 추적한다:

```
□ Step 1: {단계명}
□ Step 2: {단계명}
□ Step N: ...
```

---

### Step 1: {단계}

- **인풋**: {이 단계에 필요한 입력}
- **아웃풋**: {이 단계의 결과물}

{구체적 행동}

### Step 2: {단계}

- **인풋**: {이전 단계의 아웃풋}
- **아웃풋**: {이 단계의 결과물}

{구체적 행동}

## ⚠️ 핵심 원칙

### ❌ 금지

- {사항}

### ✅ 필수

- {사항}

## 기대 결과

- {결과물}
```

### 기존 Skill 참조 (패턴 확인용)

- `skills/creating-pr/` - 단순 워크플로우
- `skills/fixing-bugs/` - 단계별 references/ 활용
- `skills/developing-feature/` - 복잡한 순차적 단계

---

## Step 5: references/ 분리 (선택)

### 분리 기준

- 3개 이상 순차적 단계
- 각 단계가 독립 실행 가능
- 단계별 상세 설명이 긴 경우

### 구조

```
skills/{name}/
├── SKILL.md (개요 + 참조)
└── references/
    ├── 1_step.md
    └── 2_step.md
```

### 참조 방식

```markdown
### Step 1: 컨텍스트 수집

**목표:** 초기 정보 수집
**상세:** `references/1_step.md`
```

---

## Step 6: 검증

- [ ] `skills/{name}/SKILL.md` 생성
- [ ] name: 소문자+하이픈, Gerund 형태
- [ ] description: 트리거 조건 + 다른 Skill 구분 포함
- [ ] **Step 0: To-Do 생성 섹션 포함** (3단계 이상인 경우 필수)
- [ ] 200-300 lines 이내
- [ ] 오버스펙 체크 완료

---

## 간결성 원칙

**목표:** 200-300 lines (최대 350)

- 예시는 출력 형식만
- 설명보다 템플릿/체크리스트
- 300 lines 초과 → references/로 분리
