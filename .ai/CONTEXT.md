#### 중첩 보조 레이어 스케일링 버그 수정
- **문제**: Icon/Normal/Search (21.6x20) detach 후 108x20으로 늘어남
- **원인**: 내부 Ratio(108x20) 삭제 후에도 Shape(108x20)가 남아 프레임 확장
- **해결**:
  1. `cleanupOversizedChildren` → `cleanupAndScaleOversizedChildren` 변경
  2. 원본 크기(`originalWidth/Height`) 기준으로 모든 자식 비교
  3. 보조 레이어(Ratio, Constraints, Spacer)는 삭제
  4. 일반 레이어(Shape 등)는 비율 유지하며 스케일 다운
  5. cleanup 전 Auto Layout 해제로 HUG 방지

**수정 파일**: `src/code.ts`
- `cleanupAndScaleOversizedChildren(parent, targetWidth, targetHeight)` 함수 추가

## 중요 결정사항 (이번 세션)

1. **UI 상태 vs 비즈니스 상태 구분**
   - UI 상태 허용: Default, Pressed, Hover, Disabled, Loading, Focus
   - 비즈니스 상태 금지: Authenticated, Empty, Success

2. **터치 타겟 통일**
   - 48x48px (iOS 44px, Android 48px 중 큰 값)

3. **토큰 계층 도입**
   - Global → Semantic → Component 3단계
   - Figma 변수와 매핑 가능

4. **접근성 필수화**
   - WCAG 2.1 대비 기준
   - accessibilityLabel 필수
   - Reduce Motion 대응

5. **다크 모드 원칙**
   - 순수 검정(#000) 대신 #121212 사용
   - 그림자 대신 표면 밝기로 고도 표현
   - 채도 낮춘 피드백 색상
