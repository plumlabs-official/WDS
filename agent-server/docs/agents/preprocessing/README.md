# Preprocessing (전처리)

디자인 시스템 자동화 전에 수동으로 실행하는 정리 작업입니다.
**전체 실행에서 제외**되며, 필요에 따라 개별적으로 실행합니다.

## 기능 목록

| 기능 | 설명 | 타입 |
|------|------|------|
| [의미 없는 래퍼 제거](./cleanup-wrappers.md) | 단일 자식 프레임 제거 | Rule-based |
| [컴포넌트 브레이크](./detach-components.md) | 인스턴스 → 프레임 전환 | Rule-based |
| [꺼진 레이어 삭제](./delete-hidden-layers.md) | hidden 레이어 정리 | Rule-based + 확인 |

## 권장 실행 순서

```
1. 컴포넌트 브레이크 (필요시)
   - 기존 컴포넌트 연결 해제

2. 꺼진 레이어 삭제
   - 불필요한 숨겨진 요소 정리

3. 의미 없는 래퍼 제거
   - 레이어 구조 단순화
```

## 전체 실행에서 제외된 이유

1. **파괴적 작업**: 되돌리기 어려운 변경 (detach, delete)
2. **확인 필요**: 사용자가 결과를 검토해야 함
3. **선택적 적용**: 모든 디자인에 필요하지 않음

## 공통 주의사항

- 전처리 후 반드시 결과 확인
- 실수 시 Cmd+Z로 복구 가능
- 중요 파일은 작업 전 백업 권장
