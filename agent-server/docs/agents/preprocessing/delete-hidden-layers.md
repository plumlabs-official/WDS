# Delete Hidden Layers (꺼진 레이어 삭제)

## 개요
`visible: false` 상태의 레이어를 찾아 사용자 확인 후 삭제합니다.
불필요한 숨겨진 요소를 정리하여 파일 크기를 줄이고 구조를 단순화합니다.

## 타입
- **Rule-based** (LLM 사용 안 함)
- **사용자 확인 필수** (모달 UI)

## 동작 방식

### 1. 탐색 단계
1. 선택된 프레임의 모든 자식을 재귀적으로 탐색
2. `visible === false`인 노드 수집
3. 숨겨진 노드의 자식은 탐색하지 않음 (부모만 삭제하면 됨)

### 2. 확인 단계
1. 발견된 숨겨진 레이어를 **임시로 켜서** 화면에 표시
2. 해당 레이어들을 선택 상태로 변경
3. 뷰포트를 해당 레이어로 이동
4. 확인 모달 표시 (레이어 이름 목록)

### 3. 사용자 선택
- **삭제**: 모든 숨겨진 레이어 삭제
- **취소**: 레이어를 다시 숨김 상태로 복원

## UI 흐름
```
[꺼진 레이어 삭제 버튼 클릭]
    ↓
[숨겨진 레이어 검색]
    ↓
[발견된 레이어 켜기 + 선택 + 뷰포트 이동]
    ↓
[모달 표시: "N개의 꺼진 레이어가 발견되었습니다"]
    ↓
[삭제] → 레이어 삭제 → 완료 알림
[취소] → 레이어 다시 숨김 → 취소 알림
```

## 모달 UI
- 제목: "꺼진 레이어 삭제"
- 내용: 발견된 레이어 개수 + 이름 목록
- 버튼: [취소] [삭제]

## 주의사항
- 삭제는 되돌릴 수 없음 (Undo 가능)
- 숨겨진 레이어가 없으면 즉시 알림
- 부모가 숨겨져 있으면 자식은 별도로 수집하지 않음

## 파일 위치
- 구현: `src/code.ts` → `handleDeleteHiddenLayers()`, `confirmDeleteHiddenLayers()`, `cancelDeleteHiddenLayers()`
- UI: `src/ui.html` → 모달 및 핸들러
